# Prompt Templates

## 项目初拟

```text
我需要完成一个完整项目，用于【用途】。方向是【方向】。请给出 3 个可行项目方案，每个方案包括项目名称、目标用户、用户问题、核心功能、推荐语言、推荐技术栈、最小可运行版本、展示效果、实现难度和推荐理由。最后推荐一个最稳妥方案。
```

## 生成 PRD

```text
请为项目《【项目名称】》生成轻量级 PRD.md，包含项目名称、目标用户、用户问题、项目目标、主要用户流程、核心功能、关键约束、当前假设、最小可运行版本范围、暂不实现内容和待澄清问题。
```

## 生成设计文档

```text
请基于 PRD.md 进行需求澄清，并生成 memory-bank/design-document.md。内容包括项目背景、项目目标、范围、非目标、用户流程、功能行为、数据或状态模型、边界情况、异常处理和验收标准。
```

## 技术选型

```text
请根据 design-document.md 生成 memory-bank/tech-stack.md。明确编程语言、框架、第三方库、开发工具、运行环境、安装方式、启动方式、测试方式、替代方案和选择理由。优先简单稳定，不追求热门技术。
```

## 实现计划

```text
请根据 design-document.md 和 tech-stack.md 生成 memory-bank/implementation-plan.md。计划必须 code-free；每一步必须小、具体、有顺序、可独立执行、可独立验证，并包含步骤目标、涉及文件、具体任务、不做什么、验证方式和完成标准。
```

## 单步实现

```text
请读取 memory-bank/design-document.md、tech-stack.md、implementation-plan.md、progress.md 和 architecture.md，只执行当前实现计划中的一个步骤。写代码前说明目标、涉及文件和验证方式；完成后给出运行或测试命令；不要提前执行下一步。
```

## 单步验证

```text
当前步骤是：【粘贴步骤】。当前运行结果是：【粘贴结果】。请判断是否满足完成标准，是否需要更新 progress.md 和 architecture.md。若通过，生成更新记录；若不通过，给出最小修复方案。
```

## 报错修复

```text
项目运行时报错：【粘贴报错】。相关代码：【粘贴代码】。最近修改：【说明】。请定位报错类型、文件位置、原因，给出最小修复方案、是否需要回退、修复代码、验证命令和防复发建议。
```

## 技术报告

```text
请根据项目当前文档和代码生成技术报告，包含项目背景、目标、需求分析、技术选型、系统设计、模块设计、核心技术、实现过程、测试结果、AI 工具使用过程、项目亮点、不足与改进、总结。
```
